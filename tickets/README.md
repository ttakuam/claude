# チケット管理

このディレクトリには、開発チケットが phase-0 から phase-8 までのディレクトリに分割されて格納されています。

## ディレクトリ構造

```
tickets/
├── phase-0/  # プロジェクトセットアップ
├── phase-1/  # インフラ・環境構築
├── phase-2/  # データベース設計・構築
├── phase-3/  # 認証システム
├── phase-4/  # 基本UI・共通コンポーネント
├── phase-5/  # ユーザー向け機能
├── phase-6/  # 管理者向け機能
├── phase-7/  # テスト・最適化
└── phase-8/  # デプロイ・運用準備
```

## チケット命名規則

- ファイル名: `XXX-Y.md`
- XXX: フェーズ番号を含む3桁の数字 (例: 001, 101, 201)
- Y: サブタスク識別子 (A, B, C...)

## 開発順序

1. Phase 0 から順番に実行することを推奨
2. 各チケットには依存関係が記載されているため、依存チケットを先に完了させる必要があります

## 優先度について

各チケットには以下の優先度が設定されています:
- 🔴 High: 最優先で実施すべきタスク
- 🟡 Medium: 重要度が中程度のタスク
- 🟢 Low: 優先度が低いタスク(オプション含む)

## 工数見積

各チケットには工数見積(時間)が記載されており、実装の目安として利用できます。

## 使い方

1. 各フェーズのディレクトリ内のチケットファイルを確認
2. 依存関係を確認し、順序を守って実装
3. タスクを完了したらチェックボックスにチェック
4. 受入条件を満たしていることを確認
